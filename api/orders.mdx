The Orders API manages the full lifecycle of a restaurant order — from the moment a waiter opens a ticket at a table to delivery and payment. It also powers the dashboard's sales statistics panel. All endpoints require a valid JWT in the `Authorization: Bearer <token>` header. Role restrictions on individual endpoints are noted below.

---

## Endpoint overview

| Method | Path | Roles |
|--------|------|-------|
| `GET` | `/api/orders` | All authenticated |
| `GET` | `/api/orders/stats` | All authenticated |
| `GET` | `/api/orders/:id` | All authenticated |
| `POST` | `/api/orders` | admin, mesero, cajero |
| `PUT` | `/api/orders/:id/items` | admin, mesero |
| `PUT` | `/api/orders/:id/status` | All authenticated |
| `DELETE` | `/api/orders/:id/items/:itemId` | admin, mesero |

---

## GET /api/orders

Returns a flat array of all orders with their line items and calculated totals, sorted by creation time descending.

```bash
curl https://your-api-host/api/orders \
  -H "Authorization: Bearer <token>"
```

---

## GET /api/orders/stats

Returns aggregated sales statistics used by the management dashboard.

**Query parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `range` | `week` \| `month` | `week` (default) returns the last 7 days; `month` returns all days in the current calendar month. |

```bash
# Weekly stats (default)
curl "https://your-api-host/api/orders/stats" \
  -H "Authorization: Bearer <token>"

# Monthly stats
curl "https://your-api-host/api/orders/stats?range=month" \
  -H "Authorization: Bearer <token>"
```

### Stats response fields

<ResponseField name="weekly" type="array">
  Array of daily sales totals sorted chronologically.

  <Expandable title="weekly item">
    <ResponseField name="day" type="string">
      Abbreviated weekday name (`Lun`, `Mar`, …) for `range=week`, or the numeric day of month (`"1"`, `"2"`, …) for `range=month`.
    </ResponseField>
    <ResponseField name="sales" type="number">
      Total revenue for that day across all non-cancelled orders.
    </ResponseField>
    <ResponseField name="date" type="string">
      ISO date string `YYYY-MM-DD`. Days with no orders are filled in with `sales: 0`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topProducts" type="array">
  Up to 5 best-selling products over the last 7 days.

  <Expandable title="topProducts item">
    <ResponseField name="name" type="string">Product name.</ResponseField>
    <ResponseField name="quantity" type="number">Total units sold.</ResponseField>
    <ResponseField name="value" type="number">Total revenue generated by this product.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="today" type="object">
  <Expandable title="today object">
    <ResponseField name="orders" type="number">Number of orders created today.</ResponseField>
    <ResponseField name="sales" type="number">Total revenue today (paid and pending orders).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="categories" type="array">
  Revenue breakdown by product category for today.

  <Expandable title="categories item">
    <ResponseField name="categoria" type="string">Category name, or `"Sin categoria"` for uncategorised products.</ResponseField>
    <ResponseField name="total_ventas" type="number">Total revenue for this category today.</ResponseField>
  </Expandable>
</ResponseField>

## GET /api/orders/:id

Returns a single order with full line-item details.

```bash
curl https://your-api-host/api/orders/42 \
  -H "Authorization: Bearer <token>"
```

Returns `404` if the order does not exist.

## POST /api/orders

Creates a new order, inserts line items, optionally records a payment method, and sets the table status to `ocupada`. If a reservation exists for the same table today, it is automatically linked and marked `completada`.

**Roles:** `admin`, `mesero`, `cajero`

### Request body

<ParamField body="mesa_id" type="number" required>
  ID of the table being served. The table must exist. After creation the table's `estado` is set to `ocupada`.
</ParamField>

<ParamField body="mesero_id" type="number" required>
  ID of the waiter (usuario) responsible for this order.
</ParamField>

<ParamField body="items" type="array" required>
  One or more line items. Each element must include `producto_id` and `cantidad`. Items with missing fields are silently skipped.

  <Expandable title="item object">
    <ParamField body="producto_id" type="number" required>ID of the menu product.</ParamField>
    <ParamField body="cantidad" type="number" required>Quantity ordered. Must be ≥ 1.</ParamField>
    <ParamField body="precio_unitario" type="number">Override price. If omitted, the current product price from the database is used.</ParamField>
    <ParamField body="observaciones" type="string">Per-item notes (e.g. "sin cebolla").</ParamField>
  </Expandable>
</ParamField>

<ParamField body="observaciones" type="string">
  General notes for the entire order.
</ParamField>

<ParamField body="cliente_id" type="number">
  ID of an existing customer record to associate with this order.
</ParamField>

<ParamField body="reserva_id" type="number">
  ID of an existing reservation to link. If omitted, the server looks for any `confirmada` or `pendiente` reservation for this table today and links it automatically.
</ParamField>

<ParamField body="metodo_pago" type="string">
  Payment method. Accepted values: `efectivo`, `tarjeta`, `qr`, `transferencia`. When provided a payment record is created immediately and `estado_pago` is set to `pagado`.
</ParamField>

### Example request

```bash
curl -X POST https://your-api-host/api/orders \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "mesa_id": 3,
    "mesero_id": 5,
    "items": [
      { "producto_id": 12, "cantidad": 2 },
      { "producto_id": 7,  "cantidad": 1, "observaciones": "sin sal" }
    ],
    "observaciones": "Cumpleaños, por favor traer postre de cortesía",
    "cliente_id": 8
  }'
```

### Example response

```json
{
  "id": 101,
  "message": "Pedido creado correctamente"
}
```

## PUT /api/orders/:id/items

Adds new line items to an existing order or updates the quantity of existing ones. Existing items are identified by their `id` field; objects without `id` are inserted as new items.

**Roles:** `admin`, `mesero`

```bash
curl -X PUT https://your-api-host/api/orders/101/items \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      { "id": 55, "producto_id": 12, "cantidad": 3 },
      { "producto_id": 9, "cantidad": 1 }
    ]
  }'
```

---

## PUT /api/orders/:id/status

Updates the service status of an order. When the order transitions to `delivered` or `cancelled`, the associated table's `estado` is automatically set back to `libre`. A notification is also dispatched to the relevant staff role.

**Roles:** All authenticated

### Request body

<ParamField body="status" type="string" required>
  English status string. Accepted values and their internal equivalents:

  | English (`status`) | Spanish (`estado_servicio`) | Triggers table `libre` |
  |--------------------|----------------------------|------------------------|
  | `pending`   | `pendiente`  | No  |
  | `preparing` | `preparando` | No  |
  | `ready`     | `listo`      | No  |
  | `delivered` | `entregado`  | **Yes** |
  | `cancelled` | `cancelado`  | **Yes** |
</ParamField>

### Example request

```bash
curl -X PUT https://your-api-host/api/orders/101/status \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{ "status": "preparing" }'
```

---

## DELETE /api/orders/:id/items/:itemId

Removes a single line item from an order.

**Roles:** `admin`, `mesero`

```bash
curl -X DELETE https://your-api-host/api/orders/101/items/55 \
  -H "Authorization: Bearer <token>"
```

Returns `404` if `itemId` does not exist.

## Order response object

The following fields are returned by `GET /api/orders` and `GET /api/orders/:id`.

<ResponseField name="id" type="number">Unique order ID.</ResponseField>
<ResponseField name="mesa_id" type="number">ID of the table.</ResponseField>
<ResponseField name="mesa_numero" type="number">Human-readable table number.</ResponseField>
<ResponseField name="cliente_id" type="number | null">Linked customer ID, if any.</ResponseField>
<ResponseField name="cliente_nombre" type="string | null">Linked customer's name, if any.</ResponseField>
<ResponseField name="reserva_id" type="number | null">Linked reservation ID, if any.</ResponseField>
<ResponseField name="mesero_id" type="number">ID of the assigned waiter.</ResponseField>
<ResponseField name="mesero_nombre" type="string">Full name of the assigned waiter.</ResponseField>
<ResponseField name="estado_servicio" type="string">Current service status in Spanish: `pendiente`, `preparando`, `listo`, `entregado`, or `cancelado`.</ResponseField>
<ResponseField name="estado_pago" type="string">Payment status: `pendiente`, `pagado`, or `anulado`.</ResponseField>
<ResponseField name="metodo_pago" type="string | null">Payment method recorded, if paid.</ResponseField>
<ResponseField name="observaciones" type="string | null">General order notes.</ResponseField>
<ResponseField name="subtotal" type="number">Sum of all line-item subtotals.</ResponseField>
<ResponseField name="total" type="number">Same as `subtotal` (tax not applied at the API level).</ResponseField>
<ResponseField name="time" type="string">Creation time formatted as `HH:MM` in `es-ES` locale.</ResponseField>
<ResponseField name="creado_en" type="string">Full ISO timestamp of order creation.</ResponseField>
<ResponseField name="status" type="string">English-mapped service status: `pending`, `preparing`, `ready`, `delivered`, or `cancelled`.</ResponseField>
<ResponseField name="items" type="array">
  Line items belonging to this order.

  <Expandable title="items array">
    <ResponseField name="id" type="number">Line-item ID.</ResponseField>
    <ResponseField name="producto_id" type="number">Product ID.</ResponseField>
    <ResponseField name="nombre" type="string">Product name at time of order.</ResponseField>
    <ResponseField name="cantidad" type="number">Quantity ordered.</ResponseField>
    <ResponseField name="precio_unitario" type="number">Unit price charged.</ResponseField>
    <ResponseField name="subtotal" type="number">cantidad × precio_unitario.</ResponseField>
    <ResponseField name="estado" type="string">Item-level status.</ResponseField>
    <ResponseField name="observaciones" type="string | null">Per-item notes.</ResponseField>
  </Expandable>
</ResponseField>
