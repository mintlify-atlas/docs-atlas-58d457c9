title: "Settings API — Restaurant Configuration"
sidebarTitle: "Settings"

---

The Settings API exposes a single configuration record (row `id = 1` of the `configuracion` table) that holds the restaurant's display information: its trading name, street address, telephone number, and NIT (tax identification). This data appears in the header of every PDF report generated by the system, so keeping it accurate ensures all printed documents reflect the correct restaurant identity. If the configuration row does not yet exist, `GET /api/settings` returns an object with the default values shown below. Both endpoints require the `admin` role.

---

## Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `GET` | `/api/settings` | admin | Retrieve the current configuration |
| `PUT` | `/api/settings` | admin | Update the configuration |

---

## GET /api/settings

Returns the current restaurant configuration. If the `configuracion` table contains no row, an object with empty strings is returned rather than a 404.

```bash
curl -X GET https://api.example.com/api/settings \
  -H "Authorization: Bearer <token>"
```

**Response `200 OK`:**

```json
{
  "nombre_restaurante": "SAN SALVADOR",
  "direccion": "Av. Principal 123, Santa Cruz",
  "telefono": "+591 3 3456789",
  "nit": "1234567890"
}
```

<ResponseField name="nombre_restaurante" type="string">
  Restaurant trading name. Displayed as the title in PDF report headers. Default: `"SAN SALVADOR"`.
</ResponseField>
<ResponseField name="direccion" type="string">
  Street address printed below the name in report headers. Default: `""`.
</ResponseField>
<ResponseField name="telefono" type="string">
  Contact telephone number printed alongside the address. Default: `""`.
</ResponseField>
<ResponseField name="nit" type="string">
  Restaurant NIT (Número de Identificación Tributaria). Printed on invoices and report headers. Default: `""`.
</ResponseField>

---

## PUT /api/settings

Updates the restaurant configuration. `nombre_restaurante` is the only required field; omitting `direccion`, `telefono`, or `nit` stores an empty string for those fields.

### Request body

<ParamField body="nombre_restaurante" type="string" required>
  Restaurant trading name. Used as the primary heading in all generated PDF reports.
</ParamField>

<ParamField body="direccion" type="string">
  Street address. Appears on the second line of PDF report and invoice headers.
</ParamField>

<ParamField body="telefono" type="string">
  Contact phone number. Appears alongside the address in PDF headers.
</ParamField>

<ParamField body="nit" type="string">
  Restaurant NIT. Printed as `NIT: <value>` in PDF report headers and referenced during invoice generation.
</ParamField>

```bash
curl -X PUT https://api.example.com/api/settings \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre_restaurante": "SAN SALVADOR",
    "direccion": "Av. Principal 123, Santa Cruz",
    "telefono": "+591 3 3456789",
    "nit": "1234567890"
  }'
```

**Response `200 OK`:**

```json
{ "message": "Configuracion actualizada correctamente" }
```

### Error responses

| Status | Condition |
|--------|-----------|
| `400` | `nombre_restaurante` is missing or empty |
| `500` | Database or internal error |

---

<Note>
  Settings are read every time a PDF is generated — there is no caching. Updating the configuration takes effect immediately for all subsequent report generation requests.
</Note>

<Tip>
  The `direccion` and `telefono` fields are rendered as a single pipe-separated line in the PDF header (e.g., `Av. Principal 123 | +591 3 3456789`). If either field is empty, only the non-empty value is shown with no trailing pipe.
</Tip>
